## Aturan Wajib (ringkasan ŌĆö detail: `docs/02_RULES_GOVERNANCE.md`)

- **Fetch:** selalu `.limit(1000)` (default PostgREST 10) ┬Ę `.is('deleted_at', null)` ┬Ę scope `company_id` + role-aware (`isAllEntities=['super_admin']`, `isSalesOnly=['sales','operations']`).
- **`profiles`** pakai kolom **`active`** (TIDAK ada `deleted_at`). ┬Ę **`showToast?.(message, type)`** (urutan message dulu).
- **Soft delete** (`deleted_at`/`is_active=false`), jangan hard-delete data business.
- **DB:** GRANT setelah CREATE (tabel CLI tak auto-grant) ┬Ę trigger ordering pakai prefix `trg_z_` ┬Ę `auth.uid()` NULL di SQL Editor (test RLS di browser) ┬Ę super-admin bypass = top-level `OR is_super_admin()`.
- **Embed alias** saat constraint FK belum di-rename (`customers:accounts!sp_items_customer_id_fkey(name)`).
- **Deploy code yang berhenti baca kolom DULU, baru drop kolom.** ┬Ę Refresh `schema_snapshot.sql` via `pg_dump` setelah perubahan SQL Editor.
- **Workflow:** Inspect ŌåÆ Plan ŌåÆ Edit (scope kecil, hindari big-bang & rewrite App.jsx sekaligus) ŌåÆ Verify (`npm run build` clean, lint net-zero) ŌåÆ Summarize. Push HANYA bila diinstruksikan. Audit-before-fix untuk bug non-trivial.
- **Safety rules lengkap:** lihat `AGENTS.md` (15 non-negotiable).
